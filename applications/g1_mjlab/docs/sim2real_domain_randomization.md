# Sim2Real 域随机化 (Domain Randomization)

## 什么是域随机化

在仿真训练时，随机化物理环境参数，让策略学会适应各种条件，从而在真实世界中也能工作。

## 业界常用 DR 操作（按重要性排序）

### 几乎必做

| 操作 | 原因 | 典型范围 |
|------|------|---------|
| **摩擦系数** | 地面材质差异巨大（瓷砖/地毯/水泥） | [0.5, 1.5] |
| **质量/惯性** | 机器人个体差异、载重变化 | ±10-20% |
| **PD 增益** | 电机老化、温度影响刚度 | ±15-20% |
| **外部推力** | 真实场景有碰撞、风 | ±0.5-1.0 m/s |
| **观测噪声** | IMU 噪声、编码器精度 | 见 obs noise cfg |

### 常做

| 操作 | 原因 |
|------|------|
| **电机延迟** | 控制信号到力矩输出有 1-3 步延迟 |
| **关节阻尼** | 机械结构摩擦不确定 |
| **重心偏移** | 组装误差 |
| **地形高度扰动** | 地面不完全平整 |

### 有做但不必须

| 操作 | 场景 |
|------|------|
| 链接长度随机化 | URDF/MJCF 建模误差 |
| 传感器偏置 | 编码器零点漂移 |
| 动作延迟/丢包 | 通信不稳定 |

## 核心参数解释

### 摩擦系数

脚底和地面接触时的滑动阻力。仿真里默认是固定值（比如 0.6），但现实中瓷砖地很滑（0.3）、橡胶垫很涩（1.2）。随机化范围通常 [0.5, 1.5]，让策略不依赖特定摩擦值。

### 质量

机器人身体各部分的重量。仿真里是从 CAD 模型精确计算的，但真实机器人有制造误差、可能背了负载（比如电池不同型号重量不同）。随机化 ±10-20%，让策略适应重心变化。

### PD 增益

关节电机的控制参数：
- P（比例增益）= 给关节位置误差施加多大力矩
- D（阻尼增益）= 给关节速度施加多大阻力

仿真里设固定值，但真实电机受温度影响（冷启动 vs 运行 10 分钟后）、老化磨损、个体差异，实际刚度/阻尼跟仿真不完全一致。随机化 ±15-20%，让策略不依赖精确的电机响应。

### 简单类比

- 摩擦 → 穿不同鞋走不同地面
- 质量 → 背不同重量的背包
- PD 增益 → 关节松紧程度不同（新关节 vs 老关节）

策略如果只在固定参数下训练，换到真实环境就像穿着冰鞋站在冰面上——仿真里"完美"的动作立刻失效。DR 让策略学会在各种条件下都能走。

## 训练方式

**在每个 episode 开始时（reset），随机采样一组物理参数，整个 episode 用这组参数跑完。**

```
2048 个并行环境，每个环境各自独立：

Env 0 reset → 摩擦=0.8, 质量=35.2kg, PD_kp=21.3 → 跑 20s
Env 1 reset → 摩擦=1.1, 质量=33.8kg, PD_kp=18.7 → 跑 20s
Env 2 reset → 摩擦=0.5, 质量=36.1kg, PD_kp=22.0 → 跑 20s
...
Env 2047 reset → 摩擦=0.9, 质量=34.5kg, PD_kp=19.5 → 跑 20s
```

**同一时刻，2048 个环境有 2048 组不同的物理参数。** 策略必须用同一个网络在所有环境里都表现好——这迫使它学到对参数变化鲁棒的行为，而不是过拟合到某组特定参数。

### 关键点

- 策略**看不到**当前的物理参数值（摩擦多少、质量多少它不知道）
- 策略只能通过**观测反馈**（IMU、关节速度）间接感知环境差异
- 这就是为什么 Teacher-Student 蒸馏有用——Teacher 有 privileged obs（能看到真实物理参数），Student 只有传感器，通过历史观测推断当前环境

### 代码实现

在 mjlab 中通过 `EventTermCfg(mode="startup")` 实现——"startup" 表示环境创建时执行一次、之后每次 reset 时重新采样：

```python
"randomize_friction": EventTermCfg(
    mode="startup",
    func=dr.geom_friction,
    params={
        "asset_cfg": SceneEntityCfg("robot", geom_names=FOOT_GEOM_NAMES),
        "operation": "abs",
        "ranges": (0.6, 1.2),
        "shared_random": True,
    },
),
"randomize_mass": EventTermCfg(
    mode="startup",
    func=dr.body_mass,
    params={
        "asset_cfg": SceneEntityCfg("robot", body_names="pelvis"),
        "operation": "scale",
        "ranges": (0.9, 1.1),
    },
),
"randomize_pd_gains": EventTermCfg(
    mode="startup",
    func=dr.pd_gains,
    params={
        "asset_cfg": SceneEntityCfg("robot"),
        "kp_range": (0.85, 1.15),
        "kd_range": (0.85, 1.15),
        "operation": "scale",
    },
),
```
